### Architecture Decisions

### Key Bug Fixed
**Bun.write append silently overwrites**: `Bun.write(path, data, { append: true })` ignores the append flag in Bun 1.3.5. Every event, chat message, and inbox entry was being overwritten. Fixed with `node:fs/promises appendFile`. This was the root cause of events.jsonl only having 1 line.

## Lessons Learned
- **Pattern**: `Bun.write(path, data, { append: true })` silently ignores append in Bun 1.3.5. Always use `node:fs/promises appendFile` for JSONL files.
- **Mistake**: Kept calling Telegram `getUpdates` manually while debugging CCBot, which consumed the very updates I was looking for. Observer effect.
- **Discovery**: The user's electronics analogy (negative feedback loop) maps perfectly to system self-healing: sensor (heartbeat), error signal (PULSE event), comparator (dispatch rules), actuator (auto-fix), verification (next heartbeat cycle).
- **Pattern**: `drop_pending_updates: true` throws away messages on every restart. Only use during initial setup, never in production.
- **Mistake**: Had maw.js installed and working but spent hours building workarounds instead of testing it. "Try before plan."
