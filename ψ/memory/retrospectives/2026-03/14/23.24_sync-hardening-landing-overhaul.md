## 💭 Honest Feedback

This session demonstrated both the power and the chaos of heavy parallel agent orchestration. At peak, 7 agents were running simultaneously — impressive throughput but it burned through rate limits. The user had to point out visual bugs I should have caught proactively.

The Tier 1 sync work was clean and surgical. The landing page work was messy — too many incremental patches instead of one thoughtful overhaul. The user is right to demand a complete rebuild rather than patchwork.

### Friction Points
1. **Rate limit exhaustion**: Running 7+ parallel agents is powerful but unsustainable. Need to be more strategic about agent allocation — use Haiku for research, save Opus for synthesis and code.
2. **Missing prerequisite checks**: Launched image gen without GEMINI_API_KEY, Cloudflare deploy without auth. Wasted compute and time. Always verify prerequisites first.
3. **Reactive vs proactive QA**: User caught dark mode contrast, duplicate sections, oversized images. These should have been flagged in the initial audit or quick wins pass. Need a self-review step before declaring work "done."

## Lessons Learned
- **Pattern**: Check API keys/auth BEFORE spawning agents that need them — saves time and rate limits
- **Pattern**: Stripe/Linear use CSS mockups, not AI images — more reliable, no API dependency
- **Discovery**: SHA256(station|badge|timestamp) is collision-proof for idempotency keys across reinstalls
- **Pattern**: Dark mode needs token-based design system, not color inversion — desaturate accents 20-40%
