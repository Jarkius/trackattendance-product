## 💭 Honest Feedback

This session covered an enormous amount of ground in 70 minutes — MQTT debugging, 4 repo explorations, commercial planning, repo creation, PR merging, 3 releases, and workspace restructuring. The breadth was impressive but also meant each piece got less attention than it might deserve.

The /deep-research fix was satisfying but fragile — MQTT broker management on macOS with homebrew services is a recurring pain point. Two instances running because one was started by launchd and another manually. There should be a startup check.

The Gemini Deep Research integration worked perfectly once MQTT was fixed, but getting the response back required multiple attempts with retained message confusion. The extension publishes responses with retain=true, so stale responses linger. A "clear retained + wait for fresh" pattern should be standard.

The 12 parallel Haiku agents for /learn were effective but the output was massive. Distilling 50K+ tokens of agent output into concise docs is the bottleneck. The quality gate (main agent reviews + writes) is important but time-consuming.

### Friction Points
1. **Stale MQTT retained messages**: Getting the Gemini response required clearing retained messages first — should be automated in the deep-research script
2. **Symlink cycles in ψ/learn copy**: `cp -r` follows symlinks and hits cycles — need `rsync --no-links` or just don't copy ψ/learn/origin/ dirs
3. **Working directory mismatch**: Entire session ran from archive path while all work was in office/projects-office — confusing for file operations

## Lessons Learned
- **Pattern**: Duplicate system services (mosquitto, redis, etc.) are a common silent failure — always check `ps aux | grep` before debugging connectivity
- **Discovery**: Per-station pricing is a massive competitive advantage over per-user models for large workforce deployments
- **Insight**: The biggest competitor for TrackAttendance in SEA isn't Jibble or Clockify — it's paper and punch cards
