## 📝 AI Diary

This was the most ambitious orchestration session yet. Three Claude agents running simultaneously via MAW — each modifying different files in the same repo. The coordination was imperfect (agents occasionally overwrote each other's work, context exhaustion after ~150K tokens) but the velocity was undeniable: 1,420 lines of production code in 2 minutes.

The landing page went through multiple design iterations. The user has strong design opinions — they wanted Deloitte green, then switched to purple, then wanted light theme. Each change required a full color sweep across 80+ references. The lesson: establish the color palette BEFORE building 1,000 lines of HTML. I should have asked upfront.

The antigravity-kit discovery was valuable. It's not just UI components — it's a full agent/skill/workflow system with UX psychology, color theory, typography guides. Promoting 8 skills to Oracle's global set means every future project benefits from professional design knowledge.

Gemini continued to deliver Thai-specific insights that I can't match: "เช็คอินหน้างาน" vs "สแกนเช็คชื่อ" is the difference between sounding professional and sounding like a school attendance system. The multi-AI workflow (Gemini for market/language, Claude for code/architecture) is proving to be the optimal pattern.

The MAW web UI still has issues (session detaching, fleet view rendering) but the CLI (`maw peek`, `maw hey`) works perfectly. The web UI is a nice-to-have, not a blocker.

## 💭 Honest Feedback

The session was incredibly productive but chaotic. Managing 3 MAW agents + Gemini + landing page design simultaneously stretched my coordination ability. I was often dispatching tasks to agents without checking if previous tasks completed properly — leading to some overwrites and lost work.

The user's feedback loop is fast — they see results immediately and course-correct. This is valuable but means I need to be more responsive to style preferences early (color, font, terminology) before building 1,000 lines.

The antigravity-kit integration was smooth and immediately useful. The UX psychology skill alone justified the install. But 38 skills + 20 agents is a lot of context to load — we should only load what's needed per task.

### Friction Points
1. **Agent context exhaustion**: MAW agents ran out of context after ~150K tokens, becoming unresponsive. Need to either reset agents or give shorter, focused tasks.
2. **Color palette iteration**: Changed from cyan → green → purple, requiring 3 full sweeps of 80+ color references each time. Should lock colors before building.
3. **MAW web UI session detaching**: Clicking agent cards in the web UI detaches the terminal session, showing a black screen. Users can't easily monitor agents via web.

## Lessons Learned
- **Pattern**: MAW multi-agent works best for parallel, non-overlapping tasks. Give each agent a separate file/directory.
- **Mistake**: Iterating colors 3 times on a 1,000-line file wastes 15+ minutes of sed commands. Lock the palette first.
- **Discovery**: Antigravity-kit's frontend-design skill has UX psychology laws (Hick's, Fitts', Von Restorff) that make design decisions systematic, not subjective.
