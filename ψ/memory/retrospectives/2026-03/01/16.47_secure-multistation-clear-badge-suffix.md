### Architecture Decisions
- **clear_epoch as timestamp comparison**: Not a flag to clear, but a version marker. Stations compare their local epoch to cloud — only clear when they differ. Inert once matched.
- **First-time station safety**: New station initializes local epoch to cloud value without clearing — prevents accidental data loss on first connection.
- **Heartbeat piggybacking**: Station status reported during existing 60s health check cycle — zero additional network overhead.
- **Station name is configuration, not data**: Preserved across all clear operations. Never requires re-entry.
- **Atomic clear+heartbeat**: Within a single health check cycle, station detects mismatch → clears → heartbeats as "ready". Never shows as "pending" on status view.

## Lessons Learned
