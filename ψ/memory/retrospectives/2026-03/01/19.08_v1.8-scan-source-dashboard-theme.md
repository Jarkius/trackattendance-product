### Key Code Changes
- **Fastify schema**: Added `scan_source: { type: ["string", "null"], maxLength: 32 }` to batch endpoint — this was THE fix for scan_source always being "manual" in cloud
- **Light theme**: Class-based (`html.light`) with `prefers-color-scheme` detection and localStorage toggle (auto → light → dark cycle)
- **Sorting**: `ORDER BY CASE WHEN ... = 'Unmatched' THEN 1 ELSE 0 END, name ASC` across all 4 dashboard queries
- **Station visibility**: Removed `WHERE last_seen_at > NOW() - INTERVAL '5 minutes'` — offline stations now persist until explicitly cleared

### Architecture Decisions
- **Theme toggle over media query only**: User wanted manual override, not just system preference. Class-based approach allows three states (auto/light/dark) vs binary media query
- **No heartbeat TTL**: Clear operations (truncate/delete) handle cleanup, so no time-based filtering needed. Simpler, fewer edge cases
- **Red dot for offline**: More actionable than grey — admin immediately sees which stations need attention

## Lessons Learned
- **Pattern**: Fastify `additionalProperties: false` silently strips unlisted fields — always verify schema properties match your payload
- **Pattern**: CSS custom property architecture enables theme switching with minimal code — just override the variables
- **Discovery**: Clear operations (truncate/delete) can serve as the cleanup mechanism for associated tables, eliminating need for TTL-based purging
- **Pattern**: Three-state theme toggle (auto/light/dark) with localStorage is a clean UX pattern for respecting system preference while allowing override
