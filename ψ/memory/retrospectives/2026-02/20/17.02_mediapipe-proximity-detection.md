### Key Code Changes

- **ProximityDetector**: Complete rewrite from motion-based to MediaPipe face+pose
- **Model loading**: Uses `mp.tasks.vision.FaceDetector` and `mp.tasks.vision.PoseLandmarker`
- **Detection flow**: Face first (faster), pose fallback, motion as last resort
- **Resource management**: Added `close()` method for proper cleanup
- **Confidence tuning**: Lowered from 0.5 to 0.3 for better detection range

### Architecture Decisions

- **Bundled models**: Included TFLite files in repo (~6MB) rather than download at runtime — ensures offline operation
- **Tasks API over solutions**: MediaPipe 0.10.x deprecates `mp.solutions`, must use `mp.tasks`
- **Face-first detection**: Face detection is faster than pose, so try it first
- **Graceful fallback**: If MediaPipe unavailable, falls back to motion detection automatically

## AI Diary

This session was a fascinating exercise in adapting to API evolution. When I initially wrote the MediaPipe integration, I used the familiar `mp.solutions.face_detection` pattern from older tutorials and documentation. But when running tests, I hit `module 'mediapipe' has no attribute 'solutions'` — a clear sign that MediaPipe 0.10.x had moved on.

The discovery moment was satisfying: `dir(mediapipe)` revealed `['Image', 'ImageFormat', 'tasks']` instead of the expected `solutions`. The new tasks API is cleaner but requires model files, which meant downloading TFLite models and understanding the new initialization pattern with `BaseOptions` and task-specific options.

What surprised me was how much better the detection worked once I lowered the confidence threshold from 0.5 to 0.3. The initial live test showed 0 detections, which seemed broken. But a direct single-frame test revealed the face was detected at 0.32 confidence — just below threshold. A small config tweak made everything work beautifully: 8 face detections over 15 seconds, perfectly spaced by the 2-second cooldown.

The user's request for a "mini screen" demo was exactly the right instinct — seeing green bounding boxes and magenta pose skeletons in real-time makes the system tangible and testable.
