### Friction Points
1. **Nested repo confusion**: Working directory was in frontend but git commands hit the root repo initially. Need to always verify which repo we're operating in before git operations.
2. **pythonnet platform dependency**: No platform-conditional install in requirements.txt. A `requirements-dev-mac.txt` or conditional in pip would prevent this friction for future Mac developers.
3. **No API key for testing**: The placeholder key lets the app start, but we couldn't verify the full cloud sync flow. Need the real key for meaningful integration testing.

## Lessons Learned
- **Pattern**: PyQt6 apps with `.env`-driven config are highly portable across platforms for UI development
- **Discovery**: `pythonnet`/`clr_loader` are the only Windows-specific dependencies in this stack
- **Practice**: Always confirm which git repo you're in when working with nested repo structures
