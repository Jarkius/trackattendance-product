### Architecture Decisions
- Used `TRUNCATE TABLE` instead of `DELETE FROM` for cloud clear — faster, resets sequences
- Required `X-Confirm-Delete: DELETE ALL SCANS` header — prevents accidental API calls
- Admin button hidden by default, only visible when `ADMIN_PIN` is set in .env
- Local BU breakdown trades multi-station aggregate for speed — acceptable since each station has its own local DB
- Debounce values: 200ms for sync status (rapid scans), 5s for connection checks (window focus events)

## Honest Feedback

This session was highly productive — 10 files changed across 2 repos, deployed to production, all in under an hour. The plan mode workflow worked exceptionally well for a multi-file, multi-repo change. Having the task list kept execution focused and measurable.

The biggest efficiency gain was from the explore agents. Instead of reading files one by one, three parallel agents mapped out the entire dashboard flow in about 30 seconds. This meant the plan was based on actual code analysis, not assumptions.

One area of friction was the gcloud authentication. Non-interactive terminals can't handle OAuth flows, so the user had to switch to their terminal to complete auth. This is a recurring pattern with CLI tools that need browser-based auth — worth noting for future sessions.

### Friction Points
1. **gcloud auth in non-interactive terminal**: The `--no-launch-browser` flag still needs stdin for the verification code. Had to fall back to user running it manually. Impact: ~2 minutes lost.
2. **Neon CLI org selection**: `neon projects list` prompts interactively for org selection even when outputting JSON. Needed `--org-id` flag to work non-interactively. Impact: minor, but shows CLI tool assumptions about TTY.
3. **Node.js pg module not installed**: API directory had no `node_modules` — needed `npm install` before running ad-hoc queries. Could have used the deployed API endpoints instead of trying local DB connections.

## Lessons Learned
- **Pattern**: Local SQLite queries are dramatically faster than cloud API round-trips for dashboard data — even with no indexes, local is sub-millisecond vs 1-2 seconds for API calls
- **Discovery**: The `X-Confirm-Delete` header pattern is an effective safety mechanism for destructive endpoints — it can't be triggered accidentally by browsers or simple curl commands
- **Pattern**: Debouncing event-triggered functions (focus, visibility, online) prevents thundering-herd queries when users alt-tab rapidly between windows
